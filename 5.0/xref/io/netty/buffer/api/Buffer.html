28  /**
29   * A life cycled buffer of memory, with separate reader and writer offsets.
30   * <p>
31   * A buffer is a logically sequential stretch of memory with a certain capacity, an offset for writing,
32   * and an offset for reading.
33   * Buffers may be {@linkplain CompositeBuffer composed} of multiple {@linkplain #countComponents() components},
34   * where each component is a guaranteed contiguous chunk of memory.
35   *
36   * <h3>Creating a buffer</h3>
37   *
38   * Buffers are created by {@linkplain BufferAllocator allocators}, and their {@code allocate} family of methods.
39   * A number of standard allocators exist, and are available through static methods on the {@code BufferAllocator}
40   * interface.
41   *
42   * <h3>Buffer life cycle</h3>
43   *
44   * The buffer has a life cycle, where it is allocated, used, and deallocated.
45   * When the buffer is initially allocated, a pairing {@link #close()} call will deallocate it.
46   * If a buffer is {@linkplain #send() sent} elsewhere, the {@linkplain #close() close} method on the given instance
47   * will become a no-op.
48   * The buffer can be thought of as a view onto memory, and calling {@link #send()} on the buffer will effectively close
49   * that view, and recreate it upon reception at its destination.
50   *
51   * <h3>Thread-safety</h3>
52   *
53   * Buffers are <strong>not</strong> thread-safe.
54   *
55   * <h3>Accessing data</h3>
56   *
57   * Data access methods fall into two classes:
58   * <ol>
59   *     <li>Access that are based on, and updates, the read or write offset positions.</li>
60   *     <ul><li>These accessor methods are typically called {@code readX} or {@code writeX}.</li></ul>
61   *     <li>Access that take offsets as arguments, and do not update read or write offset positions.</li>
62   *     <ul><li>These accessor methods are typically called {@code getX} or {@code setX}.</li></ul>
63   * </ol>
64   *
65   * A buffer contains two mutable offset positions: one for reading and one for writing.
66   * These positions use <a href="https://en.wikipedia.org/wiki/Zero-based_numbering">zero-based indexing</a>,
67   * such that the first byte of data in the buffer is placed at offset {@code 0},
68   * and the last byte in the buffer is at offset {@link #capacity() capacity - 1}.
69   * The {@link #readerOffset()} is the offset into the buffer from which the next read will take place,
70   * and is initially zero.
71   * The reader offset must always be less than or equal to the {@link #writerOffset()}.
72   * The {@link #writerOffset()} is likewise the offset into the buffer where the next write will take place.
73   * The writer offset is also initially zero, and must be less than or equal to the {@linkplain #capacity() capacity}.
74   * <p>
75   * This carves the buffer into three regions, as demonstrated by this diagram:
76   * <pre>
77   *      +-------------------+------------------+------------------+
78   *      | discardable bytes |  readable bytes  |  writable bytes  |
79   *      |                   |     (CONTENT)    |                  |
80   *      +-------------------+------------------+------------------+
81   *      |                   |                  |                  |
82   *      0      <=     readerOffset  <=   writerOffset    <=    capacity
83   * </pre>
84   *
85   * <h3>Byte Order</h3>
86   *
87   * Buffers are always big endian, and this cannot be changed.
88   * Usages that need to get, set, read, or write, little-endian values will have to flip the byte order of the values
89   * they read and write.
90   *
91   * <h3 name="split">Splitting buffers</h3>
92   *
93   * The {@link #split()} method breaks a buffer into two.
94   * The two buffers will share the underlying memory, but their regions will not overlap, ensuring that the memory is
95   * safely shared between the two.
96   * <p>
97   * Splitting a buffer is useful for when you want to hand over a region of a buffer to some other,
98   * perhaps unknown, piece of code, and relinquish your ownership of that buffer region in the process.
99   * Examples include aggregating messages into an accumulator buffer, and sending messages down the pipeline for
100  * further processing, as split buffer regions, once their data has been received in its entirety.
101  *
102  * If you instead wish to temporarily share a region of a buffer, you will have to pass offset and length along with the
103  * buffer, or you will have to make a copy of the region.
104  *
105  * <h3>Buffers as constants</h3>
106  *
107  * Sometimes, the same bit of data will be processed or transmitted over and over again. In such cases, it can be
108  * tempting to allocate and fill a buffer once, and then reuse it.
109  * Such reuse must be done carefully, however, to avoid a number of bugs.
110  * The {@link BufferAllocator} has a {@link BufferAllocator#constBufferSupplier(byte[])} method that solves this, and
111  * prevents these bugs from occurring.
112  */
113 public interface Buffer extends Resource<Buffer>, BufferAccessor {
114     /**
115      * The capacity of this buffer, that is, the maximum number of bytes it can contain.
116      *
117      * @return The capacity in bytes.
118      */
119     int capacity();

121     /**
122      * Get the current reader offset. The next read will happen from this byte offset into the buffer.
123      *
124      * @return The current reader offset.
125      */
126     int readerOffset();
127 
128     /**
129      * Set the reader offset to {@code readerOffset() + delta}.
130      *
131      * @param delta to accumulate.

133      * {@link #writerOffset()}.
134      * @throws BufferClosedException if this buffer is closed.
135      */
136     void skipReadable(int delta);
137 
138     /**
139      * Set the reader offset. Make the next read happen from the given offset into the buffer.
140      *
141      * @param offset The reader offset to set.
142      * @return This Buffer.
143      * @throws IndexOutOfBoundsException if the specified {@code offset} is less than zero or greater than the current
144      *                                   {@link #writerOffset()}.
145      * @throws BufferClosedException if this buffer is closed.
146      */
147     Buffer readerOffset(int offset);
148 
149     /**
150      * Get the current writer offset. The next write will happen at this byte offset into the buffer.
151      *
152      * @return The current writer offset.
153      */
154     int writerOffset();
155 
156     /**
157      * Set the write offset to {@code writerOffset() + delta}.
158      *
159      * @param delta to accumulate.
160      * @throws IndexOutOfBoundsException if the new writer offset is less than the current {@link #readerOffset()} or
161      * greater than {@link #capacity()}.
162      * @throws BufferClosedException if this buffer is closed.
163      * @throws BufferReadOnlyException if this buffer is {@linkplain #readOnly() read-only}.
164      */
165     void skipWritable(int delta);
166 
167     /**
168      * Set the writer offset. Make the next write happen at the given offset.
169      *
170      * @param offset The writer offset to set.
171      * @return This Buffer.
172      * @throws IndexOutOfBoundsException if the specified {@code offset} is less than the current
173      * {@link #readerOffset()} or greater than {@link #capacity()}.
174      * @throws BufferClosedException if this buffer is closed.
175      * @throws BufferReadOnlyException if this buffer is {@linkplain #readOnly() read-only}.
176      */
177     Buffer writerOffset(int offset);

179     /**
180      * Returns the number of readable bytes which is equal to {@code (writerOffset() - readerOffset())}.
181      */
182     int readableBytes();
183 
184     /**
185      * Returns the number of writable bytes which is equal to {@code (capacity() - writerOffset())}.
186      */

189     /**
190      * Fills the buffer with the given byte value. This method does not respect the {@link #readerOffset()} or {@link
191      * #writerOffset()}, but copies the full capacity of the buffer. The {@link #readerOffset()} and {@link
192      * #writerOffset()} are not modified.
193      *
194      * @param value The byte value to write at every offset in the buffer.
195      * @return This Buffer.
196      * @throws BufferReadOnlyException if this buffer is {@linkplain #readOnly() read-only}.

200     /**
201      * Makes this buffer read-only. This is irreversible.
202      * This operation is also idempotent, so calling this method multiple times on the same buffer makes no difference.
203      *
204      * @return This buffer instance.
205      */
206     Buffer makeReadOnly();

208     /**
209      * Queries if this buffer is read-only or not.
210      *
211      * @return {@code true} if this buffer is read-only, {@code false} otherwise.
212      */
213     boolean readOnly();
214 
215     /**
216      * Queries if this buffer is backed by native memory, or not.
217      *
218      * @return {@code true} if this buffer is backed by native, off-heap, memory. Otherwise, {@code false}, if this
219      * buffer is backed by on-heap memory.
220      */
221     boolean isDirect();

223     /**
224      * Copies the given length of data from this buffer into the given destination array, beginning at the given source
225      * position in this buffer, and the given destination position in the destination array.
226      * <p>
227      * This method does not read or modify the {@linkplain #writerOffset() write offset} or the
228      * {@linkplain #readerOffset() read offset}.
229      *
230      * @param srcPos The byte offset into this buffer from where the copying should start; the byte at this offset in
231      *              this buffer will be copied to the {@code destPos} index in the {@code dest} array.
232      * @param dest The destination byte array.
233      * @param destPos The index into the {@code dest} array from where the copying should start.
234      * @param length The number of bytes to copy.
235      * @throws NullPointerException if the destination array is null.
236      * @throws IndexOutOfBoundsException if the source or destination positions, or the length, are negative,
237      * or if the resulting end positions reaches beyond the end of either this buffer, or the destination array.
238      * @throws BufferClosedException if this buffer is closed.
239      */
240     void copyInto(int srcPos, byte[] dest, int destPos, int length);
241 
242     /**
243      * Copies the given length of data from this buffer into the given destination byte buffer, beginning at the given
244      * source position in this buffer, and the given destination position in the destination byte buffer.
245      * <p>
246      * This method does not read or modify the {@linkplain #writerOffset() write offset} or the
247      * {@linkplain #readerOffset() read offset}, nor is the position of the destination buffer changed.
248      * <p>
249      * The position and limit of the destination byte buffer are also ignored, and do not influence {@code destPos}
250      * or {@code length}.
251      *
252      * @param srcPos The byte offset into this buffer from where the copying should start; the byte at this offset in
253      *              this buffer will be copied to the {@code destPos} index in the {@code dest} array.
254      * @param dest The destination byte buffer.
255      * @param destPos The index into the {@code dest} array from where the copying should start.
256      * @param length The number of bytes to copy.
257      * @throws NullPointerException if the destination buffer is null.
258      * @throws IndexOutOfBoundsException if the source or destination positions, or the length, are negative,
259      * or if the resulting end positions reaches beyond the end of either this buffer, or the destination array.
260      * @throws java.nio.ReadOnlyBufferException if the destination buffer is read-only.
261      * @throws BufferClosedException if this buffer is closed.
262      */
263     void copyInto(int srcPos, ByteBuffer dest, int destPos, int length);
264 
265     /**
266      * Copies the given length of data from this buffer into the given destination buffer, beginning at the given
267      * source position in this buffer, and the given destination position in the destination buffer.
268      * <p>
269      * This method does not read or modify the {@linkplain #writerOffset() write offset} or the
270      * {@linkplain #readerOffset() read offset} on this buffer, nor on the destination buffer.
271      * <p>
272      * The read and write offsets of the destination buffer are also ignored, and do not influence {@code destPos}
273      * or {@code length}.
274      *
275      * @param srcPos The byte offset into this buffer from where the copying should start; the byte at this offset in
276      *              this buffer will be copied to the {@code destPos} index in the {@code dest} array.
277      * @param dest The destination buffer.
278      * @param destPos The index into the {@code dest} array from where the copying should start.
279      * @param length The number of bytes to copy.
280      * @throws NullPointerException if the destination buffer is null.
281      * @throws IndexOutOfBoundsException if the source or destination positions, or the length, are negative,
282      * or if the resulting end positions reaches beyond the end of either this buffer, or the destination array.
283      * @throws BufferReadOnlyException if the destination buffer is read-only.
284      * @throws BufferClosedException if this or the destination buffer is closed.
285      */
286     void copyInto(int srcPos, Buffer dest, int destPos, int length);

288     /**
289      * Read from this buffer and write to the given channel.
290      * The number of bytes actually written to the channel are returned.
291      * No more than the given {@code length} of bytes, or the number of {@linkplain #readableBytes() readable bytes},
292      * will be written to the channel, whichever is smaller.
293      * If the channel has a position, then it will be advanced by the number of bytes written.
294      * The {@linkplain #readerOffset() reader-offset} of this buffer will likewise be advanced by the number of bytes
295      * written.
296      *
297      * @implNote {@linkplain CompositeBuffer composite buffers} may offer an optimized implementation of this method,
298      * if the given channel implements {@link GatheringByteChannel}.
299      *
300      * @param channel The channel to write to.
301      * @param length The maximum number of bytes to write.
302      * @return The actual number of bytes written, possibly zero.
303      * @throws IOException If the write-operation on the channel failed for some reason.
304      */
305     int transferTo(WritableByteChannel channel, int length) throws IOException;
306 
307     /**
308      * Read from the given channel and write to this buffer.
309      * The number of bytes actually read from the channel are returned, or -1 is returned if the channel has reached
310      * the end-of-stream.
311      * No more than the given {@code length} of bytes, or the number of {@linkplain #writableBytes() writable bytes},
312      * will be read from the channel, whichever is smaller.
313      * If the channel has a position, then it will be advanced by the number of bytes read.
314      * The {@linkplain #writerOffset() writer-offset} of this buffer will likewise be advanced by the number of bytes
315      * read.
316      *
317      * @implNote {@linkplain CompositeBuffer composite buffers} may offer an optimized implementation of this method,
318      * if the given channel implements {@link ScatteringByteChannel}.
319      *
320      * @param channel The channel to read from.
321      * @param length The maximum number of bytes to read.
322      * @return The actual number of bytes read, possibly zero, or -1 if the end-of-stream has been reached.
323      * @throws IOException If the read-operation on the channel failed for some reason.
324      */
325     int transferFrom(ReadableByteChannel channel, int length) throws IOException;

327     /**
328      * Writes into this buffer, all the bytes from the given {@code source} using the passed {@code charset}.
329      * This updates the {@linkplain #writerOffset() write offset} of this buffer.
330      *
331      * @param source {@link CharSequence} to read from.
332      * @param charset {@link Charset} to use for writing.
333      * @return This buffer.
334      */
335     default Buffer writeCharSequence(CharSequence source, Charset charset) {
336         Statics.writeCharSequence(source, this, charset);
337         return this;
338     }

340     /**
341      * Reads a {@link CharSequence} of the passed {@code length} using the passed {@link Charset}.
342      * This updates the {@linkplain #readerOffset()} reader offset} of this buffer.
343      *
344      * @param length of {@link CharSequence} to read.
345      * @param charset of the bytes to be read.
346      * @return {@link CharSequence} read from this buffer.
347      * @throws IndexOutOfBoundsException if the passed {@code length} is more than the {@linkplain #readableBytes()} of
348      * this buffer.
349      */
350     default CharSequence readCharSequence(int length, Charset charset) {
351         return Statics.readCharSequence(this, length, charset);
352     }
353 
354     /**
355      * Writes into this buffer, all the readable bytes from the given buffer.
356      * This updates the {@linkplain #writerOffset() write offset} of this buffer, and the
357      * {@linkplain #readerOffset() reader offset} of the given buffer.
358      *
359      * @param source The buffer to read from.
360      * @return This buffer.
361      * @throws NullPointerException If the source buffer is {@code null}.
362      */
363     default Buffer writeBytes(Buffer source) {
364         int size = source.readableBytes();
365         int woff = writerOffset();
366         source.copyInto(source.readerOffset(), this, woff, size);
367         source.readerOffset(source.readerOffset() + size);
368         writerOffset(woff + size);
369         return this;
370     }
371 
372     /**
373      * Writes into this buffer, all the bytes from the given byte array.
374      * This updates the {@linkplain #writerOffset() write offset} of this buffer by the length of the array.
375      *
376      * @param source The byte array to read from.
377      * @return This buffer.
378      */
379     default Buffer writeBytes(byte[] source) {
380         return writeBytes(source, 0, source.length);
381     }
382 
383     /**
384      * Writes into this buffer, the given number of bytes from the byte array.
385      * This updates the {@linkplain #writerOffset() write offset} of this buffer by the length argument.
386      *
387      * @param source The byte array to read from.
388      * @param srcPos Position in the {@code source} from where bytes should be written to this buffer.
389      * @param length The number of bytes to copy.
390      * @return This buffer.
391      */
392     default Buffer writeBytes(byte[] source, int srcPos, int length) {
393         int woff = writerOffset();
394         writerOffset(woff + length);
395         for (int i = 0; i < length; i++) {
396             setByte(woff + i, source[srcPos + i]);
397         }
398         return this;
399     }
400 
401     /**
402      * Read from this buffer, into the destination array, the given number of bytes.
403      * This updates the {@linkplain #readerOffset() read offset} of this buffer by the length argument.
404      *
405      * @param destination The byte array to write into.
406      * @param destPos Position in the {@code destination} to where bytes should be written from this buffer.
407      * @param length The number of bytes to copy.
408      * @return This buffer.
409      */
410     default Buffer readBytes(byte[] destination, int destPos, int length) {
411         int roff = readerOffset();
412         copyInto(roff, destination, destPos, length);
413         readerOffset(roff + length);
414         return this;
415     }

417     /**
418      * Resets the {@linkplain #readerOffset() read offset} and the {@linkplain #writerOffset() write offset} on this

421      * @return This buffer instance.

457     /**
458      * Opens a cursor to iterate the given number bytes of this buffer, starting at the given offset.
459      * The {@linkplain #readerOffset() reader offset} and {@linkplain #writerOffset() writer offset} are not modified by

461      * <p>
462      * Care should be taken to ensure that the buffer's lifetime extends beyond the cursor and the iteration, and that
463      * the {@linkplain #readerOffset() reader offset} and {@linkplain #writerOffset() writer offset} are not modified
464      * while the iteration takes place. Otherwise, unpredictable behaviour might result.
465      *
466      * @param fromOffset The offset into the buffer where iteration should start.
467      *                  The first byte read from the iterator will be the byte at this offset.
468      * @param length The number of bytes to iterate.
469      * @return A {@link ByteCursor} for the given stretch of bytes of this buffer.
470      * @throws IllegalArgumentException if the length is negative, or if the region given by the {@code fromOffset} and
471      * the {@code length} reaches outside the bounds of this buffer.
472      */
473     ByteCursor openCursor(int fromOffset, int length);

475     /**
476      * Opens a cursor to iterate the readable bytes of this buffer, in reverse.
477      * The {@linkplain #readerOffset() reader offset} and {@linkplain #writerOffset() writer offset} are not modified by
478      * the cursor.
479      * <p>
480      * Care should be taken to ensure that the buffer's lifetime extends beyond the cursor and the iteration, and that
481      * the {@linkplain #readerOffset() reader offset} and {@linkplain #writerOffset() writer offset} are not modified
482      * while the iteration takes place. Otherwise, unpredictable behaviour might result.
483      *
484      * @return A {@link ByteCursor} for the readable bytes of this buffer.
485      */
486     default ByteCursor openReverseCursor() {
487         int woff = writerOffset();
488         return openReverseCursor(woff == 0? 0 : woff - 1, readableBytes());
489     }

492      * Opens a cursor to iterate the given number bytes of this buffer, in reverse, starting at the given offset.
493      * The {@linkplain #readerOffset() reader offset} and {@linkplain #writerOffset() writer offset} are not modified by
494      * the cursor.
495      * <p>
496      * Care should be taken to ensure that the buffer's lifetime extends beyond the cursor and the iteration, and that
497      * the {@linkplain #readerOffset() reader offset} and {@linkplain #writerOffset() writer offset} are not modified
498      * while the iteration takes place. Otherwise, unpredictable behaviour might result.
499      *
500      * @param fromOffset The offset into the buffer where iteration should start.
501      *                  The first byte read from the iterator will be the byte at this offset.
502      * @param length The number of bytes to iterate.
503      * @return A {@link ByteCursor} for the given stretch of bytes of this buffer.
504      * @throws IllegalArgumentException if the length is negative, or if the region given by the {@code fromOffset} and
505      * the {@code length} reaches outside the bounds of this buffer.
506      */
507     ByteCursor openReverseCursor(int fromOffset, int length);

509     /**
510      * Ensures that this buffer has at least the given number of bytes of
511      * {@linkplain #writableBytes() available space for writing}.
512      * If this buffer already has the necessary space, then this method returns immediately.
513      * If this buffer does not already have the necessary space, then it will be expanded using the
514      * {@link BufferAllocator} the buffer was created with.
515      * This method is the same as calling {@link #ensureWritable(int, int, boolean)} where {@code allowCompaction} is
516      * {@code false}.
517      *
518      * @param size The requested number of bytes of space that should be available for writing.
519      * @return This buffer instance.
520      * @throws IllegalStateException if this buffer is in a bad state.
521      * @throws BufferClosedException if this buffer is closed.
522      * @throws BufferReadOnlyException if this buffer is {@linkplain #readOnly() read-only}.
523      */
524     default Buffer ensureWritable(int size) {
525         ensureWritable(size, 1, true);
526         return this;
527     }

529     /**
530      * Ensures that this buffer has at least the given number of bytes of
531      * {@linkplain #writableBytes() available space for writing}.
532      * If this buffer already has the necessary space, then this method returns immediately.
533      * If this buffer does not already have the necessary space, then space will be made available in one or all of
534      * the following available ways:
535      *
536      * <ul>
537      *     <li>
538      *         If {@code allowCompaction} is {@code true}, and sum of the read and writable bytes would be enough to
539      *         satisfy the request, and it (depending on the buffer implementation) seems faster and easier to compact
540      *         the existing buffer rather than allocation a new buffer, then the requested bytes will be made available
541      *         that way. The compaction will not necessarily work the same way as the {@link #compact()} method, as the
542      *         implementation may be able to make the requested bytes available with less effort than is strictly
543      *         mandated by the {@link #compact()} method.
544      *     </li>
545      *     <li>
546      *         Regardless of the value of the {@code allowCompaction}, the implementation may make more space available
547      *         by just allocating more or larger buffers. This allocation would use the same {@link BufferAllocator}
548      *         that this buffer was created with.
549      *     </li>
550      *     <li>
551      *         If {@code allowCompaction} is {@code true}, then the implementation may choose to do a combination of
552      *         compaction and allocation.
553      *     </li>
554      * </ul>
555      *
556      * @param size The requested number of bytes of space that should be available for writing.
557      * @return This buffer instance.
558      * @param minimumGrowth The minimum number of bytes to grow by. If it is determined that memory should be allocated
559      *                     and copied, make sure that the new memory allocation is bigger than the old one by at least
560      *                     this many bytes. This way, the buffer can grow by more than what is immediately necessary,
561      *                     thus amortising the costs of allocating and copying.
562      * @param allowCompaction {@code true} if the method is allowed to modify the

616     /**
617      * Splits the buffer into two, at the {@code offset} from the current {@linkplain #readerOffset()} reader offset}
618      * position.
619      * <p>
620      * The region of this buffer that contain the previously read and readable bytes till the {@code offset}, will be
621      * captured and returned in a new buffer, that will hold its own ownership of that region. This allows the returned
622      * buffer to be independently {@linkplain #send() sent} to other threads.
623      * <p>
624      * The returned buffer will change its {@link #readerOffset()} to {@code readerOffset() + offset}, and have its
625      * {@link #writerOffset()} and {@link #capacity()} both set to the {@code offset}.
626      * <p>
627      * The memory region in the returned buffer will become inaccessible through this buffer. This buffer will have its
628      * capacity reduced by the capacity of the returned buffer, read offset will become zero and relative position of
629      * write offset will be preserved from the provided {@code offset}, even though their position in memory remain
630      * unchanged.
631      * <p>
632      * Effectively, the following transformation takes place:
633      * <pre>{@code
634      *         This buffer:
635      *          +------------------------------------------+
636      *         0|   |r/o    offset        |w/o             |cap
637      *          +---+---------+-----------+----------------+
638      *         /   /         / \          \                \
639      *        /   /         /   \          \                \
640      *       /   /         /     \          \                \
641      *      /   /         /       \          \                \
642      *     /   /         /         \          \                \
643      *    +---+---------+           +----------+----------------+
644      *    |   |r/o      |w/o & cap  |r/o      w/o               |cap
645      *    +---+---------+           +---------------------------+
646      *    Returned buffer.                   This buffer.
647      * }</pre>
648      * When the buffers are in this state, both of the split parts retain an atomic reference count on the
649      * underlying memory. This means that shared underlying memory will not be deallocated or returned to a pool, until
650      * all the split parts have been closed.
651      * <p>
652      * Composite buffers have it a little easier, in that at most only one of the constituent buffers will actually be
653      * split. If the split point lands perfectly between two constituent buffers, then a composite buffer can
654      * simply split its internal array in two.
655      * <p>
656      * Split buffers support all operations that normal buffers do, including {@link #ensureWritable(int)}.
657      * <p>
658      * See the <a href="#split">Splitting buffers</a> section for details.
659      *
660      * @return A new buffer with independent and exclusive ownership over the previously read and readable bytes from
661      * this buffer.
662      */
663     default Buffer readSplit(int offset) {
664         return split(readerOffset() + offset);
665     }

768      * Splits the buffer into two, at the given {@code splitOffset}.

787      *         /               / \               \
788      *        /               /   \               \
789      *       /               /     \               \
790      *      /               /       \               \
791      *     /               /         \               \
792      *    +---------------+           +---------------+
793      *    |               |cap        |               |cap
794      *    +---------------+           +---------------+
795      *    Returned buffer.            This buffer.
796      * }</pre>

798      * underlying memory. This means that shared underlying memory will not be deallocated or returned to a pool, until
799      * all the split parts have been closed.

801      * Composite buffers have it a little easier, in that at most only one of the constituent buffers will actually be
802      * split. If the split point lands perfectly between two constituent buffers, then a composite buffer can
803      * simply split its internal array in two.

806      * <p>

861      * <p>
862      * The given {@linkplain ReadableComponentProcessor processor} is called for each readable component in this buffer,
863      * and passed a component index, for the given component in the iteration, and a {@link ReadableComponent} object
864      * for accessing the data within the given component.
865      * <p>
866      * The component index is specific to the particular invocation of this method. The first call to the consumer will
867      * be passed the given initial index, and the next call will be passed the initial index plus one, and so on.
868      * <p>
869      * The {@linkplain ReadableComponentProcessor component processor} may stop the iteration at any time by returning
870      * {@code false}.
871      * This will cause the number of components processed to be returned as a negative number (to signal early return),
872      * and the number of components processed may then be less than the
873      * {@linkplain #countReadableComponents() readable component count}.
874      * <p>

876      * multiple calls, so the data must be extracted from the component in the context of the iteration.

879      * This means they can be accessed as long as the internal memory store remain unchanged. Methods that may cause

881      * {@link #ensureWritable(int, int, boolean)}, and {@link #send()}.

886      * should not be used for changing the buffer contents. Doing so may cause undefined behaviour.

889      * this buffer instance.
890      *
891      * @param initialIndex The initial index of the component for iteration, and the index that will be passed to the
892      * first call to the {@linkplain ReadableComponentProcessor#process(int, ReadableComponent) processor}.
893      * @param processor The processor that will be used to process the buffer components.
894      * @return The number of readable components processed, as a positive number if all readable components were
895      * processed, or as a negative number if the iteration was stopped because
896      * {@link ReadableComponentProcessor#process(int, ReadableComponent)} returned {@code false}.
897      * In any case, the number of components processed may be less than {@link #countComponents()}.
898      */
