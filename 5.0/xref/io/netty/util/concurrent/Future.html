18  import java.util.concurrent.CancellationException;
19  import java.util.concurrent.CompletionException;
20  import java.util.concurrent.ExecutionException;
21  import java.util.concurrent.TimeUnit;
22  import java.util.concurrent.TimeoutException;
23  import java.util.function.Function;

25  /**
26   * The result of an asynchronous operation.
27   * <p>
28   * An asynchronous operation is one that might be completed outside a given thread of execution. The operation can
29   * either be performing computation, or I/O, or both.
30   * <p>
31   * All I/O operations in Netty are asynchronous. It means any I/O calls will return immediately with no guarantee that
32   * the requested I/O operation has been completed at the end of the call. Instead, you will be returned with a {@link
33   * Future} instance which gives you the information about the result or status of the I/O operation.
34   * <p>
35   * A {@link Future} is either <em>uncompleted</em> or <em>completed</em>. When an I/O operation begins, a new future
36   * object is created. The new future is uncompleted initially - it is neither succeeded, failed, nor cancelled because
37   * the I/O operation is not finished yet. If the I/O operation is finished either successfully, with failure, or by
38   * cancellation, the future is marked as completed with more specific information, such as the cause of the failure.
39   * Please note that even failure and cancellation belong to the completed state.
40   * <pre>
41   *                                      +---------------------------+
42   *                                      | Completed successfully    |
43   *                                      +---------------------------+
44   *                                 +---->      isDone() = true      |
45   * +--------------------------+    |    |   isSuccess() = true      |
46   * |        Uncompleted       |    |    +===========================+
47   * +--------------------------+    |    | Completed with failure    |
48   * |      isDone() = false    |    |    +---------------------------+
49   * |   isSuccess() = false    |----+---->      isDone() = true      |
50   * | isCancelled() = false    |    |    |       cause() = non-null  |
51   * |       cause() = throws   |    |    +===========================+
52   * |      getNow() = throws   |    |    | Completed by cancellation |
53   * +--------------------------+    |    +---------------------------+
54   *                                 +---->      isDone() = true      |
55   *                                      | isCancelled() = true      |
56   *                                      +---------------------------+
57   * </pre>
58   * <p>
59   * Various methods are provided to let you check if the I/O operation has been completed, wait for the completion, and
60   * retrieve the result of the I/O operation. It also allows you to add {@link FutureListener}s so you can get notified
61   * when the I/O operation is completed.
62   *
63   * <h3>Prefer {@link #addListener(FutureListener)} to {@link #await()}</h3>
64   * <p>
65   * It is recommended to prefer {@link #addListener(FutureListener)}, or {@link #addListener(Object,
66   * FutureContextListener)}, to {@link #await()} wherever possible to get notified when an I/O operation is done and to
67   * do any follow-up tasks.
68   * <p>
69   * The {@link #addListener(FutureListener)} method is non-blocking. It simply adds the specified {@link FutureListener}
70   * to the {@link Future}, and the I/O thread will notify the listeners when the I/O operation associated with the future
71   * is done. The {@link FutureListener} and {@link FutureContextListener} callbacks yield the best performance and
72   * resource utilization because it does not block at all, but it could be tricky to implement a sequential logic if you
73   * are not used to event-driven programming.
74   * <p>
75   * By contrast, {@link #await()} is a blocking operation. Once called, the caller thread blocks until the operation is
76   * done. It is easier to implement a sequential logic with {@link #await()}, but the caller thread blocks unnecessarily
77   * until the I/O operation is done and there's relatively expensive cost of inter-thread notification. Moreover, there's
78   * a chance of dead-lock in a particular circumstance, which is described below.
79   *
80   * <h3>Do not call {@link #await()} inside a {@link io.netty5.channel.ChannelHandler}</h3>
81   * <p>
82   * The event handler methods in {@link io.netty5.channel.ChannelHandler} are usually called by an I/O thread. If {@link
83   * #await()} is called by an event handler method, which is called by the I/O thread, the I/O operation it is waiting
84   * for might never complete because {@link #await()} can block the I/O operation it is waiting for, which is a
85   * dead-lock.
86   * <pre>
87   * // BAD - NEVER DO THIS
88   * {@code @Override}
89   * public void channelRead({@link io.netty5.channel.ChannelHandlerContext} ctx, Object msg) {
90   *     {@link Future} future = ctx.channel().close();
91   *     future.awaitUninterruptibly();
92   *     // Perform post-closure operation
93   *     // ...
94   * }
95   *
96   * // GOOD
97   * {@code @Override}
98   * public void channelRead({@link io.netty5.channel.ChannelHandlerContext} ctx, Object msg) {
99   *     {@link Future} future = ctx.channel().close();
100  *     future.addListener(new {@link FutureListener}() {
101  *         public void operationComplete({@link Future} future) {
102  *             // Perform post-closure operation
103  *             // ...
104  *         }
105  *     });
106  * }
107  * </pre>
108  * <p>
109  * In spite of the disadvantages mentioned above, there are certainly the cases where it is more convenient to call
110  * {@link #await()}. In such a case, please make sure you do not call {@link #await()} in an I/O thread. Otherwise,
111  * {@link BlockingOperationException} will be raised to prevent a dead-lock.
112  *
113  * <h3>Do not confuse I/O timeout and await timeout</h3>
114  * <p>
115  * The timeout value you specify with {@link #await(long)}, {@link #await(long, TimeUnit)}, {@link
116  * #awaitUninterruptibly(long)}, or {@link #awaitUninterruptibly(long, TimeUnit)} are not related with I/O timeout at
117  * all.  If an I/O operation times out, the future will be marked as 'completed with failure,' as depicted in the
118  * diagram above.  For example, connect timeout should be configured via a transport-specific option:
119  * <pre>
120  * // BAD - NEVER DO THIS
121  * {@link io.netty5.bootstrap.Bootstrap} b = ...;
122  * {@link Future} f = b.connect(...);
123  * f.awaitUninterruptibly(10, TimeUnit.SECONDS);
124  * if (f.isCancelled()) {
125  *     // Connection attempt cancelled by user
126  * } else if (!f.isSuccess()) {
127  *     // You might get a NullPointerException here because the future
128  *     // might not be completed yet.
129  *     f.cause().printStackTrace();
130  * } else {
131  *     // Connection established successfully
132  * }
133  *
134  * // GOOD
135  * {@link io.netty5.bootstrap.Bootstrap} b = ...;
136  * // Configure the connect timeout option.
137  * <b>b.option({@link io.netty5.channel.ChannelOption}.CONNECT_TIMEOUT_MILLIS, 10000);</b>
138  * {@link Future} f = b.connect(...);
139  * f.awaitUninterruptibly();
140  *
141  * // Now we are sure the future is completed.
142  * assert f.isDone();
143  *
144  * if (f.isCancelled()) {
145  *     // Connection attempt cancelled by user
146  * } else if (!f.isSuccess()) {
147  *     f.cause().printStackTrace();
148  * } else {
149  *     // Connection established successfully
150  * }
151  * </pre>
152  */
153 public interface Future<V> extends AsynchronousResult<V> {
154     /**
155      * Adds the specified listener to this future. The specified listener is notified when this future is {@linkplain
156      * #isDone() done}. If this future is already completed, the specified listener is notified immediately.
157      *
158      * @param listener The listener to be called when this future completes. The listener will be passed this future as
159      *                 an argument.
160      * @return this future object.
161      */
162     Future<V> addListener(FutureListener<? super V> listener);
163 
164     /**
165      * Adds the specified listener to this future. The specified listener is notified when this future is {@linkplain
166      * #isDone() done}. If this future is already completed, the specified listener is notified immediately.
167      *
168      * @param context  The context object that will be passed to the listener when this future completes.
169      * @param listener The listener to be called when this future completes. The listener will be passed the given
170      *                 context, and this future.
171      * @return this future object.
172      */
173     <C> Future<V> addListener(C context, FutureContextListener<? super C, ? super V> listener);
174 
175     /**
176      * Waits for this future until it is done, and rethrows the cause of the failure if this future failed.
177      *
178      * @throws CancellationException if the computation was cancelled
179      * @throws CompletionException   if the computation threw an exception.
180      * @throws InterruptedException  if the current thread was interrupted while waiting
181      */
182     Future<V> sync() throws InterruptedException;
183 
184     /**
185      * Waits for this future until it is done, and rethrows the cause of the failure if this future failed.
186      *
187      * @throws CancellationException if the computation was cancelled
188      * @throws CompletionException   if the computation threw an exception.
189      */
190     Future<V> syncUninterruptibly();
191 
192     /**
193      * Waits for this future to be completed.
194      *
195      * @throws InterruptedException if the current thread was interrupted
196      */
197     Future<V> await() throws InterruptedException;
198 
199     /**
200      * Waits for this future to be completed without interruption.  This method catches an {@link InterruptedException}
201      * and discards it silently.
202      */
203     Future<V> awaitUninterruptibly();
204 
205     /**
206      * Waits for this future to be completed within the specified time limit.
207      *
208      * @return {@code true} if and only if the future was completed within the specified time limit
209      * @throws InterruptedException if the current thread was interrupted
210      */
211     boolean await(long timeout, TimeUnit unit) throws InterruptedException;
212 
213     /**
214      * Waits for this future to be completed within the specified time limit.
215      *
216      * @return {@code true} if and only if the future was completed within the specified time limit
217      * @throws InterruptedException if the current thread was interrupted
218      */
219     boolean await(long timeoutMillis) throws InterruptedException;
220 
221     /**
222      * Waits for this future to be completed within the specified time limit without interruption.  This method catches
223      * an {@link InterruptedException} and discards it silently.
224      *
225      * @return {@code true} if and only if the future was completed within the specified time limit
226      */
227     boolean awaitUninterruptibly(long timeout, TimeUnit unit);
228 
229     /**
230      * Waits for this future to be completed within the specified time limit without interruption.  This method catches
231      * an {@link InterruptedException} and discards it silently.
232      *
233      * @return {@code true} if and only if the future was completed within the specified time limit
234      */
235     boolean awaitUninterruptibly(long timeoutMillis);
236 
237     /**
238      * Get the result of this future, if it has completed.
239      * If the future has failed, then an {@link ExecutionException} will be thrown instead.
240      * If the future has not yet completed, then this method will block until it completes.
241      *
242      * @return The result of the task execution, if it completed successfully.
243      * @throws InterruptedException If the call was blocked, waiting for the future to complete, and the thread was
244      * {@linkplain Thread#interrupt() interrupted}.
245      * @throws ExecutionException If the task failed, either by throwing an exception or through cancellation.
246      */
247     default V get() throws InterruptedException, ExecutionException {
248         await();
249 
250         Throwable cause = cause();
251         if (cause == null) {
252             return getNow();
253         }
254         if (cause instanceof CancellationException) {
255             throw (CancellationException) cause;
256         }
257         throw new ExecutionException(cause);
258     }
259 
260     /**
261      * Get the result of this future, if it has completed.
262      * If the future has failed, then an {@link ExecutionException} will be thrown instead.
263      * If the future has not yet completed, then this method will block, waiting up to the given timeout for the future
264      * to complete.
265      * If the future does not complete within the specified timeout, then a {@link TimeoutException} will be thrown.
266      * If the timeout is zero, then this method will not block, and instead either get the result or failure of the
267      * future if completed, or immediately throw a {@link TimeoutException} if not yet completed.
268      *
269      * @param timeout The non-negative maximum amount of time, in terms of the given time unit, to wait for the
270      *                completion of the future.
271      * @param unit The time unit for the timeout.
272      * @return The value of the successfully completed future.
273      * @throws InterruptedException If this call was blocking and this thread got
274      * {@linkplain Thread#interrupt() interrupted}.
275      * @throws ExecutionException If the task failed, either by throwing an exception, or through cancellation.
276      * @throws TimeoutException If the future did not complete within the specified timeout.
277      */
278     default V get(long timeout, TimeUnit unit) throws InterruptedException, ExecutionException, TimeoutException {
279         if (await(timeout, unit)) {
280             Throwable cause = cause();
281             if (cause == null) {
282                 return getNow();
283             }
284             if (cause instanceof CancellationException) {
285                 throw (CancellationException) cause;
286             }
287             throw new ExecutionException(cause);
288         }
289         throw new TimeoutException();
290     }
291 
292     /**
293      * Returns a {@link FutureCompletionStage} that reflects the state of this {@link Future} and so will receive all
294      * updates as well.
295      */
296     default FutureCompletionStage<V> asStage() {
297         return new DefaultFutureCompletionStage<>(this);
298     }
299 
300     /**
301      * Returns a {@link java.util.concurrent.Future JDK Future that reflects the state of this {@link Future}.
302      */
303     default java.util.concurrent.Future<V> asJdkFuture() {
304         return new DefaultFutureCompletionStage<>(this);
305     }
306 
307     /**
308      * Creates a <strong>new</strong> {@link Future} that will complete with the result of this {@link Future} mapped
309      * through the given mapper function.
310      * <p>
311      * If this future fails, then the returned future will fail as well, with the same exception. Cancellation of either
312      * future will cancel the other. If the mapper function throws, the returned future will fail, but this future will
313      * be unaffected.
314      *
315      * @param mapper The function that will convert the result of this future into the result of the returned future.
316      * @param <R>    The result type of the mapper function, and of the returned future.
317      * @return A new future instance that will complete with the mapped result of this future.
318      */
319     default <R> Future<R> map(Function<V, R> mapper) {
320         return Futures.map(this, mapper);
321     }
322 
323     /**
324      * Creates a <strong>new</strong> {@link Future} that will complete with the result of this {@link Future}
325      * flat-mapped through the given mapper function.
326      * <p>
327      * The "flat" in "flat-map" means the given mapper function produces a result that itself is a future-of-R, yet this
328      * method also returns a future-of-R, rather than a future-of-future-of-R. In other words, if the same mapper
329      * function was used with the {@link #map(Function)} method, you would get back a {@code Future<Future<R>>}. These
330      * nested futures are "flattened" into a {@code Future<R>} by this method.
331      * <p>
332      * Effectively, this method behaves similar to this serial code, except asynchronously and with proper exception and
333      * cancellation handling:
334      * <pre>{@code
335      * V x = future.sync().getNow();
336      * Future<R> y = mapper.apply(x);
337      * R result = y.sync().getNow();
338      * }</pre>
339      * <p>
340      * If the given future fails, then the returned future will fail as well, with the same exception. Cancellation of
341      * either future will cancel the other. If the mapper function throws, the returned future will fail, but this
342      * future will be unaffected.
343      *
344      * @param mapper The function that will convert the result of this future into the result of the returned future.
345      * @param <R>    The result type of the mapper function, and of the returned future.
346      * @return A new future instance that will complete with the mapped result of this future.
347      */
348     default <R> Future<R> flatMap(Function<V, Future<R>> mapper) {
349         return Futures.flatMap(this, mapper);
350     }
351 
352     /**
353      * Link the {@link Future} and {@link Promise} such that if the {@link Future} completes the {@link Promise}
354      * will be notified. Cancellation is propagated both ways such that if the {@link Future} is cancelled
355      * the {@link Promise} is cancelled and vice-versa.
356      *
357      * @param promise   the {@link Promise} which will be notified
358      * @return          itself
359      */
360     default Future<V> cascadeTo(final Promise<? super V> promise) {
361         Futures.cascade(this, promise);
362         return this;
363     }
364 }
