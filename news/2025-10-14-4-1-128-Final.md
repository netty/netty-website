We are happy to announce the release of netty 4.1.128.Final. This is a bug-fix release but also contains [CVE-2025-59419](https://github.com/netty/netty/security/advisories/GHSA-jq43-27x9-3v86) which might affect you if you make use
of our SMTP implementation.

The most important changes are:

* SMTP Command Injection Vulnerability Allowing Email Forgery ([CVE-2025-59419](https://github.com/netty/netty/security/advisories/GHSA-jq43-27x9-3v86))
* Drop unknown frame on missing stream in first packet ([#15647](https://github.com/netty/netty/pull/15647))
* Precompute segments offsets and use them as segment's identity ([#15656](https://github.com/netty/netty/pull/15656))
* Empty chunks cannot be used while allocating from the shared queue ([#15659](https://github.com/netty/netty/pull/15659))
* Only register chunk sizes in adaptive allocator ([#15663](https://github.com/netty/netty/pull/15663))
* Fix concurrent chunk data write bug in adaptive allocator ([#15664](https://github.com/netty/netty/pull/15664))
* Update jni-util version to clarify licensing ([#15689](https://github.com/netty/netty/pull/15689))
* Fix Snappy compression bug ([#15715](https://github.com/netty/netty/pull/15715))
* Fix aligned off-heap zeroing ([#15744](https://github.com/netty/netty/pull/15744))

For more details please visit our [bug tracker](https://github.com/netty/netty/issues?q=milestone%3A4.1.127.Final+is%3Aclosed)
