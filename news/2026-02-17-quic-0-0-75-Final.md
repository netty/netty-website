We are happy to announce the release of netty-incubator-codec-quic 0.0.75.Final. This is a bug-fix release but also add some new advanced features.

The most important changes are:

* Upgrade quiche version to get fix for bug that can cause packet drops ([#831](https://github.com/netty/netty-incubator-codec-quic/pull/831))
* Include correct license informations for BoringSSL ([#824](https://github.com/netty/netty-incubator-codec-quic/pull/824))
* Update jni-util version to clarify licensing ([#825](https://github.com/netty/netty-incubator-codec-quic/pull/825))
* Implement Quic stream reset frame application protocol error code propagation ([#842](https://github.com/netty/netty-incubator-codec-quic/pull/842))
* Add support for quiche_config_discover_pmtu ([#839](https://github.com/netty/netty-incubator-codec-quic/pull/839))
* Fix codec-native-quic Bundle-SymbolicNames ([#834](https://github.com/netty/netty-incubator-codec-quic/pull/834))
* Add missing return which could lead to double notification of a promise and so unexpected exception ([#845](https://github.com/netty/netty-incubator-codec-quic/pull/845))
* Fix incorrect assert when handling id to channel mappings ([#844](https://github.com/netty/netty-incubator-codec-quic/pull/844))
* Correctly return all TransportParameters ([#843](https://github.com/netty/netty-incubator-codec-quic/pull/843))
* Allow to set destination connection id when creating a client side QuicChannel ([#833](https://github.com/netty/netty-incubator-codec-quic/pull/833))

For more details related to this release see our [bug-tracker](https://github.com/netty/netty-incubator-codec-quic/issues?q=is%3Aclosed+milestone%3A0.0.75.Final).

For more details about this codec in general please read our [initial announcement](https://netty.io/news/2020/12/09/quic-0-0-1-Final.html).
