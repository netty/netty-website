We are happy to announce the release of netty 4.1.134.Final. This is a bug-fix release.

We recommend upgrading to this version to get the following fixes:

* HTTP: Re-add constructor to `HttpProxyHandler` that was removed by mistake [#16750](https://github.com/netty/netty/pull/16750)
* Marshalling: Explicit document security requirements [#16754](https://github.com/netty/netty/pull/16754)
* Pin HTTP/RTSP version + method normalization to `Locale.US` [#16770](https://github.com/netty/netty/pull/16770)
* Adaptive: Fix concurrency issue in adaptive allocator [#16778](https://github.com/netty/netty/pull/16778)
* Pin multipart `Content-Type` / `Content-Transfer-Encoding` case folding to `Locale.US` [#16784](https://github.com/netty/netty/pull/16784)
* Remove dead native declarations [#16785](https://github.com/netty/netty/pull/16785)
* Avoid re-parsing openssl key material with non-cached provider [#16791](https://github.com/netty/netty/pull/16791)
* IpFilter: Fix ClassCastException caused by IpSubnetFilter if only ipv6 rules are configured but remote peer is using ipv4 [#16822](https://github.com/netty/netty/pull/16822)
* Resolve all localhost addresses without querying DNS servers [#16820](https://github.com/netty/netty/pull/16820)
* HTTP2: Use 100 as default max concurrent streams setting [#16816](https://github.com/netty/netty/pull/16816)
* Route synchronous `onLookupComplete` exceptions via `fireExceptionCaught` [#16824](https://github.com/netty/netty/pull/16824)
* Fix MQTT decoder size check after variable header replay [#16838](https://github.com/netty/netty/pull/16838)

For more details please see the complete release notes: https://github.com/netty/netty/releases/tag/netty-4.1.134.Final
