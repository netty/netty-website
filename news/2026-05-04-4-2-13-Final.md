We are happy to announce the release of netty 4.2.13.Final. This is a bug-fix and security release that fixes numerous security issues.

We strongly recommend upgrading to this version to get the following security fixes:

* [CVE-2026-42586](https://github.com/netty/netty/security/advisories/GHSA-rgrr-p7gp-5xj7) (netty-codec-redis)
* [CVE-2026-42578](https://github.com/netty/netty/security/advisories/GHSA-45q3-82m4-75jr) (netty-handler-proxy)
* [CVE-2026-42577](https://github.com/netty/netty/security/advisories/GHSA-rwm7-x88c-3g2p) (netty-transport-native-epoll)
* [CVE-2026-42587](https://github.com/netty/netty/security/advisories/GHSA-f6hv-jmp6-3vwv) (netty-codec-http, netty-codec-http2)
* [CVE-2026-41417](https://github.com/netty/netty/security/advisories/GHSA-v8h7-rr48-vmmv) (netty-codec-http)
* [CVE-2026-42581](https://github.com/netty/netty/security/advisories/GHSA-xxqh-mfjm-7mv9) (netty-codec-http)
* [CVE-2026-42580](https://github.com/netty/netty/security/advisories/GHSA-m4cv-j2px-7723) (netty-codec-http)
* [CVE-2026-42585](https://github.com/netty/netty/security/advisories/GHSA-38f8-5428-x5cv) (netty-codec-http)
* [CVE-2026-42579](https://github.com/netty/netty/security/advisories/GHSA-cm33-6792-r9fm) (netty-codec-dns)
* [CVE-2026-42582](https://github.com/netty/netty/security/advisories/GHSA-2c5c-chwr-9hqw) (netty-codec-http3)
* [CVE-2026-42583](https://github.com/netty/netty/security/advisories/GHSA-mj4r-2hfc-f8p6) (netty-codec, netty-codec-compression)
* [CVE-2026-42584](https://github.com/netty/netty/security/advisories/GHSA-57rv-r2g8-2cj3) (netty-codec-http)
* [CVE-2026-44248](https://github.com/netty/netty/security/advisories/GHSA-jfg9-48mv-9qgx) (netty-codec-mqtt)

*Breaking Changes*

[The patch](https://github.com/netty/netty/commit/84530fa81e12dcd1d42310bb20c1385cb44128d8) for
[CVE-2026-42581](https://github.com/netty/netty/security/advisories/GHSA-xxqh-mfjm-7mv9) prohibits HTTP/1.1 requests
containing both the `Transfer-Encoding` and `Content-Length` headers, in line with RFC 9112.
Previous versions of HTTP/1.1 (RFC 7230) permitted this combination. You can restore the old behavior with the
`-Dio.netty.handler.codec.http.rfc9112TransferEncoding=false` system property or with `HttpDecoderConfig`.
Note that disabling this check may lead to request smuggling vulnerabilities.

Other significant changes are:

* Kqueue: sendfile EINTR doesn't advance offset — data duplication [#16544](https://github.com/netty/netty/pull/16544)
* Ensure the `CRYPTO_BUFFER_POOL` is also freed when we fail creating the `SSLContext` [#16545](https://github.com/netty/netty/pull/16545)
* Fix `IndexOutOfBoundsException` in `StompSubframeDecoder` on heartbeat [#16543](https://github.com/netty/netty/pull/16543)
* Avoid leak in `PemReader` on `OutOfDirectMemoryError` [#16551](https://github.com/netty/netty/pull/16551)
* Include user properties and subscription IDs in `MqttProperties#isEmpty` [#16575](https://github.com/netty/netty/pull/16575)
* Native DNS resolver: Guard against `malloc` failures [#16559](https://github.com/netty/netty/pull/16559)
* Fix parsing HTTP chunks with multiple extensions [#16579](https://github.com/netty/netty/pull/16579)
* Native transports: Correctly create pipe when `pipe2` is not supported [#16592](https://github.com/netty/netty/pull/16592)
* Fix buffer component search fast path [#16548](https://github.com/netty/netty/pull/16548)
* Quic: Correctly handle `SSL_CTX_new` failures [#16622](https://github.com/netty/netty/pull/16622)
* Make `LocalIoHandle` public [#16621](https://github.com/netty/netty/pull/16621)
* Quic: Fix shadowing of variable which leads to incorrectly handling errors [#16623](https://github.com/netty/netty/pull/16623)
* Use stream error for `maxContentLength` exceeded in `InboundHttp2ToHttpAdapter` [#16629](https://github.com/netty/netty/pull/16629)
* Fix `shutdownInput` bug in kqueue for empty `recv` buffer [#16630](https://github.com/netty/netty/pull/16630)
* Fix FFM address semantics in `directBufferAddress` [#16603](https://github.com/netty/netty/pull/16603)
* HTTP2: Ensure HTTP2 preface is always send as first message [#16636](https://github.com/netty/netty/pull/16636)
* Kqueue: Fix usage of `LOCAL_PEERPID` [#16637](https://github.com/netty/netty/pull/16637)
* Fix file descriptor reuse bug in kqueue [#16650](https://github.com/netty/netty/pull/16650)
* HTTP2: Ensure HTTP2 preface is always send as first message (also on the server) [#16667](https://github.com/netty/netty/pull/16667)
* Fix `IllegalReferenceCountException` in `AdaptiveByteBuf.deallocate()` [#16654](https://github.com/netty/netty/pull/16654)
* Add generic `FileRegion` support in io_uring stream channel [#16571](https://github.com/netty/netty/pull/16571)
* Deprecate `ObjectCleaner` and remove usage [#16685](https://github.com/netty/netty/pull/16685)
* Update to netty-tcnative 2.0.77.Final [#16687](https://github.com/netty/netty/pull/16687)
* Avoid NPE in JdkSslClientContext when TrustManagerFactory returns null [#16702](https://github.com/netty/netty/pull/16702)
* Avoid NPE in JdkSslServerContext when TrustManagerFactory returns null [#16700](https://github.com/netty/netty/pull/16700)
* Epoll: Correctly delete fd from epoll if there is nothing to handle [#16689](https://github.com/netty/netty/pull/16689)
* Epoll: Use correct initial EpollIoOps [#16728](https://github.com/netty/netty/pull/16728)

For more details please see the complete release notes: https://github.com/netty/netty/releases/tag/netty-4.2.13.Final
