We are happy to announce the release of netty 4.1.131.Final. This is a bug-fix release.

The most important changes are:

* NioDatagramChannel.block(...) does not early return on failure ([#16044](https://github.com/netty/netty/pull/16044))
* Support for AWS Libcrypto (AWS-LC) netty-tcnative build ([#16081](https://github.com/netty/netty/pull/16081))
* codec-dns: Decompress MX RDATA exchange domain names during DNS record decoding ([#16127](https://github.com/netty/netty/pull/16127))
* Buddy allocation for large buffers in adaptive allocator ([#16132](https://github.com/netty/netty/pull/16132))
* SslHandler: Only resume on EventLoop if EventLoop is not shutting down already ([#16149](https://github.com/netty/netty/pull/16149))
* Wrap ECONNREFUSED in PortUnreachableException for UDP ([#16150](https://github.com/netty/netty/pull/16150))
* Bump com.ning:compress-lzf (4.1) ([#16159](https://github.com/netty/netty/pull/16159))
* Fix adaptive allocator bug from not noticing failed allocation ([#16202](https://github.com/netty/netty/pull/16202))
* Avoid loosing original read exception ([#16204](https://github.com/netty/netty/pull/16204))
* Backport multiple adaptive allocator changes ([#16219](https://github.com/netty/netty/pull/16219))
For more details please visit our [bug tracker](https://github.com/netty/netty/issues?q=milestone%3A4.1.131.Final+is%3Aclosed)
