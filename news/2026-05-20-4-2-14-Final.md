We are happy to announce the release of netty 4.2.14.Final. This is a bug-fix release.

We recommend upgrading to this version to get the following fixes:

* HTTP: Re-add constructor to `HttpProxyHandler` that was removed by mistake [#16747](https://github.com/netty/netty/pull/16747)
* Marshalling: Explicit document security requirements [#16752](https://github.com/netty/netty/pull/16752)
* Fix `io_uring` op completion `TRACE` logging [#16755](https://github.com/netty/netty/pull/16755)
* Quic: Ensure writes are done before notify close promise of `QuicheQuicChannel` [#16758](https://github.com/netty/netty/pull/16758)
* Avoid re-parsing openssl key material with non-cached provider [#16759](https://github.com/netty/netty/pull/16759)
* Pin HTTP/RTSP version + method normalization to `Locale.US` [#16765](https://github.com/netty/netty/pull/16765)
* Fill `MsgHdrMemoryArray#hdrs` with null entry on release [#16764](https://github.com/netty/netty/pull/16764)
* Adaptive: Fix concurrency issue in adaptive allocator [#16767](https://github.com/netty/netty/pull/16767)
* Make bulk byte moving in `ByteBuf` faster [#16781](https://github.com/netty/netty/pull/16781)
* Pin multipart `Content-Type` / `Content-Transfer-Encoding` case folding to `Locale.US` [#16768](https://github.com/netty/netty/pull/16768)
* Remove dead native declarations [#16783](https://github.com/netty/netty/pull/16783)
* Enable `OpenSslCachingKeyMaterialProvider` to evict stale entries after cert rotation [#16523](https://github.com/netty/netty/pull/16523)
* IoUring: extend user data from `short` to `long` [#16682](https://github.com/netty/netty/pull/16682)
* Revert `CompositeByteBuf` component search fast path [#16811](https://github.com/netty/netty/pull/16811)
* HTTP2: Use 100 as default max concurrent streams setting [#16804](https://github.com/netty/netty/pull/16804)
* Fix `ResumptionController` wrapping [#16815](https://github.com/netty/netty/pull/16815)
* Resolve all localhost addresses without querying DNS servers [#16749](https://github.com/netty/netty/pull/16749)
* IpFilter: Fix `ClassCastException` caused by `IpSubnetFilter` if only ipv6 rules are configured but remote peer is using ipv4 [#16803](https://github.com/netty/netty/pull/16803)
* Fix `memoryAddress()` for direct `ByteBuffers` wrapped by `Unpooled` without `Unsafe` [#16788](https://github.com/netty/netty/pull/16788)
* Route synchronous `onLookupComplete` exceptions via `fireExceptionCaught` [#16794](https://github.com/netty/netty/pull/16794)
* IoUring: Stop generic `FileRegion` drain loop when `transferred()` reaches `count()` [#16826](https://github.com/netty/netty/pull/16826)
* MQTT: Allow MQTT 5 CONNECT with password only [#16833](https://github.com/netty/netty/pull/16833)
* Fix MQTT decoder size check after variable header replay [#16787](https://github.com/netty/netty/pull/16787)

For more details please see the [complete release notes](https://github.com/netty/netty/releases/tag/netty-4.2.14.Final).
