We are happy to announce the release of netty 4.2.10.Final, which is our first release this year. This is a bug-fix release which also fixes a regression introduced by 4.2.9.Final so please consider upgrading as soon as possible.


The most important changes are:

* Reimplement IORING_OP_SENDMSG_ZC ([#16130](https://github.com/netty/netty/pull/16130))
* codec-dns: Decompress MX RDATA exchange domain names during DNS record decoding ([#16040](https://github.com/netty/netty/pull/16040))
* Buddy allocation for large buffers in adaptive allocator ([#16053](https://github.com/netty/netty/pull/16053))
* Improve adaptive allocator thread local performance ([#15741](https://github.com/netty/netty/pull/15741))
* Avoid UnsupportedOperationException in CompositeeByteBuf when noUnsafe is true ([#16097](https://github.com/netty/netty/pull/16097))
* NioDatagramChannel.block(...) does not early return on failure ([#16044](https://github.com/netty/netty/pull/16044))
* Set default maxFieldSectionSize to 16MB ([#16088](https://github.com/netty/netty/pull/16088))
* io_uring: submit send over write for sockets ([#16095](https://github.com/netty/netty/pull/16095))
* Limit QUIC stream and field section sizes to protocol max ([#16117](https://github.com/netty/netty/pull/16117)) 
* Avoid non-portable malloc-of-zero behavior ([#16055](https://github.com/netty/netty/pull/16055)) 
* Make AdaptiveByteBuf.setBytes faster ([#15736](https://github.com/netty/netty/pull/15736)) 
* allocation free AdaptiveByteBuf::setBytes(byte[]) ([#16058](https://github.com/netty/netty/pull/16058)) 
* fix: prevent DGC init failure if IpMulticastAll cannot be set ([#16104](https://github.com/netty/netty/pull/16104)) 
* Improve chunk picking for the large-size buddy allocator ([#16179](https://github.com/netty/netty/pull/16179)) 
* Fix adaptive allocator bug from not noticing failed allocation ([#16200](https://github.com/netty/netty/pull/16200)) 
* Avoid loosing original read exception ([#16199](https://github.com/netty/netty/pull/16199)) 
* Fix possible ClassCastException when using ReadOnlyAbstractByteBuf ([#16188](https://github.com/netty/netty/pull/16188)) 
* Update to Quiche 0.24.8 ([#16182](https://github.com/netty/netty/pull/16182)) 
* Fix incorrect assert when handling id to channel mappings ([#16163](https://github.com/netty/netty/pull/16163)) 
* Correctly return all TransportParameters ([#16162](https://github.com/netty/netty/pull/16162)) 
* Reduce allocations on DefaultHeaders::containsValue ([#15843](https://github.com/netty/netty/pull/15843)) 
* SslHandler: Only resume on EventLoop if EventLoop is not shutting down already ([#16144](https://github.com/netty/netty/pull/16144)) 
* Wrap ECONNREFUSED in PortUnreachableException for UDP ([#16131](https://github.com/netty/netty/pull/16131)) 
* Ensure ChannelPromise that are used by ChannelOutboundHandler will always complete in the assumed thread ([#16052](https://github.com/netty/netty/pull/16052)) 

For more details please visit our [bug tracker](https://github.com/netty/netty/issues?q=milestone%3A4.2.10.Final+is%3Aclosed)
