We are happy to announce the release of netty 4.1.133.Final. This is a bug-fix and security release that fixes numerous security issues.

We strongly recommend upgrading to this version to get the following security fixes:

* CVE-2026-42586 (netty-codec-redis)
* CVE-2026-42578 (netty-handler-proxy)
* CVE-2026-42587 (netty-codec-http, netty-codec-http2)
* CVE-2026-41417 (netty-codec-http)
* CVE-2026-42581 (netty-codec-http)
* CVE-2026-42580 (netty-codec-http)
* CVE-2026-42585 (netty-codec-http)
* CVE-2026-42579 (netty-codec-dns)
* CVE-2026-42582 (netty-codec-http3)
* CVE-2026-42583 (netty-codec, netty-codec-compression)
* CVE-2026-42584 (netty-codec-http)
* CVE-2026-XXXXX (netty-codec-mqtt)

Other significant changes are:

* Fix `IndexOutOfBoundsException` in `StompSubframeDecoder` on heartbeat [#16539](https://github.com/netty/netty/pull/16539)
* Kqueue: sendfile `EINTR` doesn't advance offset — data duplication [#16554](https://github.com/netty/netty/pull/16554)
* Avoid leak in `PemReader` on `OutOfDirectMemoryError` [#16576](https://github.com/netty/netty/pull/16576)
* Native DNS resolver: Guard against `malloc` failures [#16584](https://github.com/netty/netty/pull/16584)
* Include user properties and subscription IDs in `MqttProperties#isEmpty` [#16582](https://github.com/netty/netty/pull/16582)
* Fix parsing HTTP chunks with multiple extensions [#16588](https://github.com/netty/netty/pull/16588)
* Epoll: Cleanup code to always return negative value on failure [#16601](https://github.com/netty/netty/pull/16601)
* Native transports: Correctly create pipe when `pipe2` is not supported [#16598](https://github.com/netty/netty/pull/16598)
* Use stream error for `maxContentLength` exceeded in `InboundHttp2ToHttpAdapter` [#16558](https://github.com/netty/netty/pull/16558)
* Fix `shutdownInput` bug in kqueue for empty `recv` buffer [#16638](https://github.com/netty/netty/pull/16638)
* Kqueue: Fix usage of `LOCAL_PEERPID` [#16646](https://github.com/netty/netty/pull/16646)
* HTTP2: Ensure HTTP2 preface is always send as first message [#16642](https://github.com/netty/netty/pull/16642)
* HTTP2: Ensure HTTP2 preface is always send as first message (also on the server) [#16675](https://github.com/netty/netty/pull/16675)
* Deprecate `ObjectCleaner` and remove usage [#16694](https://github.com/netty/netty/pull/16694)
* Update to netty-tcnative 2.0.77.Final [#16695](https://github.com/netty/netty/pull/16695)
* Avoid NPE in JdkSslServerContext when TrustManagerFactory returns null [#16691](https://github.com/netty/netty/pull/16691)
* Avoid NPE in JdkSslClientContext when TrustManagerFactory returns null [#16690](https://github.com/netty/netty/pull/16690)
* SCTP: Correctly handle SO_BACKLOG [#16715](https://github.com/netty/netty/pull/16715)

For more details please see the complete release notes: https://github.com/netty/netty/releases/tag/netty-4.2.13.Final
