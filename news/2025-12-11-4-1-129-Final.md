We are happy to announce the release of netty 4.1.129.Final. This is a bug-fix release but also contains [CVE-2025-67735](https://github.com/netty/netty/security/advisories/GHSA-84h7-rjj3-6jx4).

The most important changes are:

* RLF injection vulnerability in io.netty.handler.codec.http.HttpRequestEncoder ([CVE-2025-67735](https://github.com/netty/netty/security/advisories/GHSA-84h7-rjj3-6jx4))
* Update lz4-java version to 1.10.1 ([#15981](https://github.com/netty/netty/pull/15981))
* Close Channel and fail bootstrap when setting a ChannelOption causes an error ([#15970](https://github.com/netty/netty/pull/15970))
* Discard the following HttpContent for preflight request ([#15962](https://github.com/netty/netty/pull/15962))
* Fix race condition in NonStickyEventExecutorGroup causing incorrect inEventLoop() results ([#15927](https://github.com/netty/netty/pull/15927))
* Fix Zstd compression for large data ([#15900](https://github.com/netty/netty/pull/15900))
* Fix ZstdEncoder not producing data when source is smaller than block ([#15894](https://github.com/netty/netty/pull/15894))
* Make big endian ASCII hashcode consistent with little endian ([#15846](https://github.com/netty/netty/pull/15846))
* Fix reentrancy bug in ByteToMessageDecoder ([#15834](https://github.com/netty/netty/pull/15834))
* Add 32k and 64k size classes to adaptive allocator ([#15800](https://github.com/netty/netty/pull/15800))
* Re-enable reflective field accesses in native images ([#15774](https://github.com/netty/netty/pull/15774))
* Correct HTTP/2 padding length check ([#15795](https://github.com/netty/netty/pull/15795))

For more details please visit our [bug tracker](https://github.com/netty/netty/issues?q=milestone%3A4.1.129.Final+is%3Aclosed)
